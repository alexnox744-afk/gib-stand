# Техническое задание: стенд расчленёнки для бумер-шутера

**Движок:** Godot 4.x
**Тип задачи:** изолированный технический стенд (sandbox)
**Цель:** проверить и отладить систему попаданий, расчленения, гибсов и рэгдолла на одном враге до интеграции в геймплей.

---

## 1. Общее описание

Сцена представляет собой платформу, на которой стоит один враг (зомби). Игрок не перемещается — камера вращается вокруг врага по орбите. Клик мышью по модели симулирует выстрел выбранным оружием в конкретную точку тела. Система должна корректно обрабатывать локальный урон по зонам, отрыв конечностей, рэгдолл при смерти и полное разнесение на куски (gib) при оверкилле.

Стенд — инструмент разработчика, а не игровая сцена. Приоритет: наглядность, отладочный UI, быстрый сброс состояния.

---

## 2. Структура сцены (ориентир)

```
TestStand (Node3D)
├── Environment
│   ├── Platform (StaticBody3D + MeshInstance3D)   # площадка, об неё бьются гибсы
│   ├── DirectionalLight3D
│   └── WorldEnvironment                            # нейтральный фон/освещение
├── EnemyPivot (Node3D)                             # точка, вокруг которой крутится камера
│   └── Enemy (Node3D)
│       ├── Skeleton3D
│       │   ├── MeshInstance3D                      # сегментированный скин-меш врага
│       │   ├── PhysicalBone3D × N                  # физический скелет для рэгдолла
│       │   └── BoneAttachment3D × N
│       │       └── Hitbox (Area3D + CollisionShape) # хитбокс зоны, метаданные зоны
│       └── HealthComponent                         # общее HP + HP по зонам
├── OrbitCamera (Node3D)                            # пивот орбиты
│   └── Camera3D
├── GibsPool (Node3D)                               # пул кусков
├── DecalPool (Node3D)                              # пул декалей крови
├── EffectsManager (Node)                           # спавн частиц/звуков
└── UI (CanvasLayer)
    ├── WeaponSelector                              # выбор оружия
    ├── HitInfoPanel                                # что и куда попало, урон
    └── DebugControls                               # сброс, тоглы опций
```

> Сегментированный меш = модель разбита на части (голова, плечи, предплечья, кисти, торс, бёдра, голени), каждую можно скрыть независимо. Это нужно для отрыва конечностей через swap на отдельный объект.

## 3. Функциональные требования

### 3.1. Орбитальная камера

- Вращение вокруг `EnemyPivot` по горизонтали и вертикали (зажатая ПКМ или СКМ + движение мыши).
- Зум колесом мыши (ограничить min/max дистанцию).
- Вертикальный угол ограничить (например, от −80° до +80°), чтобы камера не переворачивалась.
- ЛКМ зарезервирована под «выстрел» и не должна вращать камеру.

### 3.2. Клик-выстрел (raycast от курсора)

- По ЛКМ: построить луч из камеры через позицию курсора:
  - `camera.project_ray_origin(mouse_pos)` + `camera.project_ray_normal(mouse_pos)`.
  - Запрос пересечения через `PhysicsDirectSpaceState3D.intersect_ray()` по слою хитбоксов.
- Из результата получить: точку попадания, нормаль, зону (через метаданные хитбокса), направление выстрела.
- Передать эти данные в обработчик урона вместе с параметрами активного оружия.
- Поведение зависит от типа оружия (см. 3.4): один луч / несколько лучей с разбросом / попадание + сплеш.

### 3.3. Система хитбоксов и зон

- Тело разбито на **зоны урона**. Минимальный набор: `head`, `torso`, `upper_arm_L/R`, `lower_arm_L/R`, `thigh_L/R`, `shin_L/R`.
- Каждый хитбокс — `Area3D` с `CollisionShape3D` (капсула/бокс), прикреплённый к кости через `BoneAttachment3D`, чтобы следовать за анимацией.
- В метаданных хитбокса хранить: имя зоны, ссылку на сегмент меша, индекс/имя кости, можно ли зону отрывать (`severable: bool`), порог отрыва (`sever_threshold`), множитель урона (хедшот ×N).
- У каждой отрываемой зоны — собственный счётчик HP. Накопленный урон выше порога → отрыв.

### 3.4. Арсенал оружия (data-driven)

Оружие описывается ресурсом `WeaponData` (custom `Resource`), чтобы добавлять новые без правки кода. Поля:

| Поле | Назначение |
|---|---|
| `name` | отображаемое имя |
| `hit_type` | `hitscan_single` / `hitscan_spread` / `projectile_splash` / `melee` |
| `damage` | базовый урон на попадание (на дробину для spread) |
| `pellet_count` | число лучей для spread |
| `spread_angle` | конус разброса |
| `splash_radius` | радиус сплеша (для взрывных) |
| `splash_damage` | урон в эпицентре сплеша |
| `dismember_force` | импульс, придаваемый оторванной части/гибсам |
| `gib_on_direct_hit` | мгновенный gib при прямом попадании (ракета) |
| `sever_power` | насколько легко это оружие отрывает конечности |

Базовый набор для стенда (значения подберёт геймдизайн):

- **Кулак / Melee** — `melee`, низкий урон, кнокбэк, без отрыва.
- **Пистолет** — `hitscan_single`, низкий урон, отрыв только при добивании зоны.
- **Дробовик** — `hitscan_spread`, 7–9 дробин, сильный отрыв вблизи (главный «расчленитель»).
- **Чейнган / SMG** — `hitscan_single`, быстрый, средний урон.
- **Ракетница** — `projectile_splash`, `gib_on_direct_hit = true`, сплеш разносит соседние зоны.
- **Плазма** — `hitscan_single` или быстрый снаряд, средне-высокий урон.
- **Бензопила** (опц.) — `melee` непрерывный, высокий `sever_power`, фонтан крови.

> Для стенда снаряды можно делать «мгновенными» (hitscan со сплешем в точке) — реальная баллистика здесь не нужна.

### 3.5. Логика расчленения

Порядок проверки при попадании:

1. **Прямой gib (оверкилл одним ударом).** Если урон по зоне `head`/`torso` ≥ оставшегося HP врага с большим запасом, **или** `gib_on_direct_hit = true` — заменить всю модель на разлёт кусков (см. 3.6) и удалить врага. Конец обработки.
2. **Отрыв зоны.** Если зона `severable` и накопленный урон по ней ≥ `sever_threshold`:
   - скрыть сегмент меша этой зоны (и зависимые ниже по иерархии — оторвав бедро, скрыть и голень/стопу);
   - заспавнить отдельный `RigidBody3D` с моделью оторванной части (с «крышкой» среза — текстура мяса/кости на торце);
   - придать импульс по направлению выстрела × `dismember_force` + случайная угловая скорость;
   - спавн частиц крови из культи + звук + декаль;
   - отрыв головы или обоих бёдер/торса = смерть.
3. **Смерть → рэгдолл.** Если общий HP ≤ 0 (но не оверкилл):
   - `Skeleton3D.physical_bones_start_simulation()`;
   - придать импульс физкостям в направлении последнего выстрела;
   - уже оторванные зоны остаются скрытыми.
4. **Обычный урон.** Иначе — вычесть HP зоны и общий HP, проиграть фидбек попадания.

### 3.6. Гибсы (полное разнесение)

- При оверкилле: скрыть/удалить целую модель, заспавнить набор кусков (`RigidBody3D` с простой геометрией — деформированные кубы/капсулы под мясо, плюс отдельно голова целиком).
- **3–4 заготовленных пресета разлёта** на врага, выбираются рандомно → ощущение вариативности без процедурной нарезки меша.
- Импульсы: радиально от центра + вверх + случайный разброс, угловая скорость рандомная.
- Каждый кусок: спрей крови при касании пола (опц.), декаль под собой.

### 3.7. Фидбек (продаёт ощущение «фарша»)

- **Частицы крови** (`GPUParticles3D`): спрей в точке попадания вдоль нормали; облако-туман при отрыве/смерти.
- **Декали крови** (`Decal`): на платформе/стенах в точках попадания и под гибсами. Пул с лимитом, старые переиспользуются.
- **Звук:** мясной «чвак» на попадание, отдельный squelch на отрыв, «взрыв мяса» на gib, звук выстрела на оружие.
- **Камера:** лёгкий шейк на мощных попаданиях (опц., тогл в дебаг-UI).

### 3.8. Отладочный UI

- Селектор активного оружия (кнопки/дропдаун).
- Панель попадания: имя зоны, нанесённый урон, текущее HP зоны и общее HP.
- Кнопка **Reset** — мгновенно вернуть врага в исходное состояние (все сегменты на месте, HP полное, гибсы и декали очищены).
- Тоглы: показать хитбоксы (debug draw), показать имена костей, вкл/выкл шейк, вкл/выкл декали.
- Счётчики: число активных гибсов и декалей (контроль производительности).

## 4. Нефункциональные требования

- **Пулинг.** Гибсы и декали — через object pooling, не `instantiate`/`queue_free` на каждый кадр. Лимит одновременных гибсов (например, 40) и декалей (например, 60) с переиспользованием.
- **Автоудаление.** Гибс «засыпает» и удаляется в пул через 4–5 сек после остановки.
- **Упрощённый физскелет.** Симулировать только крупные кости: голова, торс, плечи, предплечья, бёдра, голени. Пальцы/мелкие кости в рэгдолл не включать.
- **Data-driven оружие.** Новое оружие = новый `.tres` ресурс, без правки кода логики.
- Стабильные 60 FPS при единичном враге со всем фаршем (стенд не нагружен толпой).

---

## 5. Этапы реализации

1. **Каркас сцены + камера.** Платформа, свет, статичная модель врага, рабочая орбитальная камера с зумом.
2. **Клик-хит + хитбоксы.** Raycast от курсора, хитбоксы по зонам, вывод зоны и точки в UI. Пока без урона.
3. **HP и обычный урон.** Система HP по зонам + общий, селектор оружия, фидбек попадания (частицы/звук/декаль).
4. **Отрыв конечностей.** Сегментация меша, swap на `RigidBody3D` оторванной части, импульсы, культи.
5. **Рэгдолл.** Физскелет, запуск симуляции на смерть, импульс от выстрела.
6. **Гибсы и оверкилл.** Пресеты разлёта, прямой gib от ракеты/добивания.
7. **Полный арсенал.** Дробовик (spread), ракетница (splash), остальное оружие, балансные значения.
8. **Полировка + пулинг.** Лимиты, автоудаление, дебаг-тоглы, оптимизация.

---

## 6. Критерии готовности (acceptance)

- Камера свободно вращается вокруг врага и зумится, ЛКМ не мешает вращению.
- Клик попадает точно в зону под курсором; зона и урон видны в UI.
- Каждое оружие из арсенала даёт визуально различимый результат (пистолет ≠ дробовик ≠ ракета).
- Конечности отрываются по порогу, с культёй, кровью и физикой оторванной части.
- Смерть без оверкилла → корректный рэгдолл без «конвульсий» (адекватные лимиты суставов).
- Оверкилл/ракета → разлёт на гибсы.
- Reset мгновенно и полностью восстанавливает врага.
- 60 FPS держится, гибсы и декали не накапливаются бесконечно.

## 7. Необходимые ресурсы

### 7.1. 3D-модели

- **Модель врага (зомби), low-poly**, с ригом (скелетом). Меш **сегментирован** по зонам отрыва (отдельные сабмеши/материалы для головы, плеч, предплечий, кистей, торса, бёдер, голеней).
- **Заготовки оторванных частей** с «крышкой» среза (рука, предплечье, кисть, нога, голень, голова) — торец закрыт геометрией с текстурой мяса/кости, чтобы не было дырки.
- **Гибсы** — набор простых кусков мяса/костей/органов (5–8 мелких low-poly мешей). Голова отдельно (отлетает целиком).
- **Платформа** — простой меш площадки.
- *(Опц.)* вьюмодели оружия — для стенда не обязательны.

### 7.2. Текстуры и материалы

- Материал/текстура врага.
- Материал среза (мясо + кость) для культей и крышек оторванных частей.
- Материал гибсов.
- Текстуры декалей крови (3–5 вариантов пятен).
- Текстура частиц крови (спрей/капли), при необходимости — для туман-облака.

### 7.3. Частицы (GPUParticles3D)

- Спрей крови (импакт по нормали).
- Облако/туман крови (отрыв и смерть).
- Бёрст при gib.

### 7.4. Звук

- Попадание по плоти (3–4 варианта).
- Отрыв конечности (squelch).
- Gib-взрыв.
- Звуки смерти зомби.
- Звук выстрела на каждое оружие.
- *(Опц.)* удары гибсов о поверхности.

### 7.5. Шейдеры (опционально)

- Шейдер декали крови (мягкие края, наложение).
- *(Опц. на будущее)* экранный эффект крови при добивании.

### 7.6. Технические заготовки кода

- `WeaponData` (custom Resource) + набор `.tres` под каждое оружие.
- `HealthComponent` (общее HP + словарь HP по зонам).
- Менеджер пулов (гибсы, декали).
- `DismembermentSystem` (логика порядка проверок из 3.5).
- Дебаг-UI (CanvasLayer со связкой на сигналы стенда).

## 8. Замечания и риски

- **Качество рэгдолла — это ручная настройка**, а не модель. Заложить время на подгонку капсул-коллайдеров и лимитов суставов на физскелете; кривые лимиты дают «бьющуюся тряпку».
- **Срезы — арт-задача.** Без подготовленных торцов оторванные части выглядят как пустые дырки. Это главный по трудозатратам арт-пункт.
- Один риг под анимацию и под физику — компромисс: чем детальнее скелет, тем дороже рэгдолл. Для зомби держать скелет умеренным.
- Стенд намеренно не тестирует толпу. Перед интеграцией в геймплей отдельно проверить производительность пулов на волне из многих врагов.
